## Overview

The curlme CLI brings the full power of curlme.io to your terminal. Stream requests in real-time, replay webhooks, diff payloads, and more—all without leaving your workflow.

## Quick Reference

### Authentication

```bash
curlme auth login      # Authenticate with API key
curlme auth whoami     # Show current user
curlme auth logout     # Remove stored credentials
```

### Bin Management

```bash
curlme bin list        # List all bins
curlme bin create      # Create a new bin
curlme bin use         # Set active bin
curlme bin info        # Show bin details
curlme bin delete      # Delete a bin
```

### Request Inspection

```bash
curlme listen          # Stream requests in real-time
curlme latest          # Show most recent request
curlme show <id>       # Inspect specific request
```

### Advanced

```bash
curlme replay <id>     # Replay request locally
curlme diff <a> <b>    # Compare two requests
curlme export          # Export request data
curlme open            # Open dashboard in browser
```

### Account & Support

```bash
curlme upgrade         # Upgrade to Pro
curlme billing         # Manage subscription
curlme feedback        # Report an issue
```

## Interactive Mode

When running `curlme listen`, use these keyboard shortcuts:

| Key | Action |
| --- | ------ |
| `Enter` | Inspect the latest request |
| `R` | Replay latest to localhost |
| `D` | Diff latest vs previous |
| `O` | Open bin in browser |
| `Ctrl+C` | Quit |

## Output Styles

The CLI uses color-coded output for easy scanning:

```
→ POST   /webhooks/stripe   200    156B  rq_f8a3b2
→ GET    /health            200     12B  rq_a1b2c3
→ POST   /webhooks/github   200    892B  rq_d4e5f6
```

| Color | Meaning |
| ----- | ------- |
| Yellow `→` | Incoming request indicator |
| Green | GET method |
| Yellow | POST/PUT/PATCH methods |
| Red | DELETE method |
| Dim | Metadata (size, timestamps) |

## Configuration

The CLI stores configuration in your system's config directory:

| OS | Path |
| -- | ---- |
| macOS | `~/Library/Preferences/curlme-cli-nodejs/` |
| Linux | `~/.config/curlme-cli-nodejs/` |
| Windows | `%APPDATA%\curlme-cli-nodejs\` |

### Stored Settings

- `apiKey` - Your authentication key
- `activeBinId` - Currently selected bin
- `baseUrl` - API endpoint (default: https://curlme.io)

## Environment Variables

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `CURLME_API_URL` | Override API endpoint | `https://curlme.io` |
